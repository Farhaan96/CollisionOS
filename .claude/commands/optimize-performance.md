# Optimize Performance

Analyze and optimize performance of a CollisionOS component or feature.

## Task
Optimize performance for: $ARGUMENTS

The workflow will:
1. Profile current performance
2. Identify bottlenecks
3. Implement optimizations
4. Measure improvements
5. Ensure functionality preserved

## Optimization Areas
- **React**: Memoization, lazy loading, virtualization
- **Database**: Query optimization, indexing
- **API**: Caching, pagination, batching
- **Bundle**: Code splitting, tree shaking
- **Assets**: Image optimization, CDN usage

## Techniques
- Use React.memo for expensive components
- Implement useMemo/useCallback hooks
- Add database indexes
- Enable query result caching
- Reduce bundle size
- Optimize render cycles

## Success Metrics
- Page load time < 2s
- Time to interactive < 3s
- Database queries < 100ms
- API response time < 500ms
- Bundle size reduced by 20%+
